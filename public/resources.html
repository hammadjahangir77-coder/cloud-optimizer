5-minute cloud audit

Five checks many small businesses skip — based on common themes in cloud cost research: budgets, unused storage, idle capacity, and schedules.

  1. 1Set budget alerts — get an email before the bill surprises you. Most providers offer this free in their billing dashboard.
  2. 2Delete unused backups — old snapshots accumulate silently. Review what you're storing and remove anything you no longer need.
  3. 3Find idle systems — servers running 24/7 with almost no traffic are pure waste. Check CPU and network utilisation graphs.
  4. 4Check orphan storage — disks not attached to any active instance still cost money. Detach and delete if safe to do so.
  5. 5Schedule off-hour shutdowns — test and training environments don't need to run nights and weekends. Auto-stop them.

Cloud billing glossary

Plain-language definitions for terms you'll see on your cloud invoice.

On-demand instance A virtual server you pay for by the hour or second with no upfront commitment.
Reserved instance A discounted rate in exchange for committing to a server size for 1 or 3 years.
Spot / preemptible Spare cloud capacity sold cheaply — can be reclaimed by the provider with short notice.
Snapshot A point-in-time backup of a disk. Old snapshots are a common source of silent spend.
Egress Data leaving the cloud (e.g. to users or another provider). Usually charged per GB.
Right-sizing Matching your server size to actual usage — switching from oversized to a smaller tier.
Idle resource Infrastructure running but doing almost no work — a key source of the 32% waste figure.
